# 什么是 A+ 内容(给运营/答疑用)

> 一句话:A+ 内容(A+ Content,旧称 EBC)是亚马逊商品详情页**商品描述区**的图文增强排版——用图片+排版模块代替纯文字描述,提升转化。
> 官方说明(需登录 Seller Central):https://sellercentral.amazon.com/help/hub/reference/GGW8U76SSNTRTBX7

## 一、是什么
- 出现在 listing 详情页的"商品描述/From the manufacturer"区域,用**模块化图文**展示卖点、对比、使用场景等。
- 由**品牌方/卖家**制作,需先完成**品牌注册(Brand Registry)** 才能使用。
- 提交后经**亚马逊审核**通过才会上线(审核通常数小时~7 天)。

## 二、两个层级
| | 基础 A+(Basic / 标准) | 高级 A+(Premium A+) |
|---|---|---|
| 模块种类 | ~17 种基础模块 | 多了视频、轮播、对比表、热点、背景大图等(共约 19 种高级模块) |
| 视觉 | 图文为主 | 更大图、视频、互动(轮播/热点)、整幅背景 |
| 获取条件 | 品牌注册即可 | 需满足额外条件(通常:品牌注册 + 已有通过的基础 A+ 记录 / 达到亚马逊的资格门槛,部分站点曾免费开放) |
| 本工具 | — | **本工具专做高级 A+**(SP-API 不支持创建高级 A+,故用浏览器自动化) |

## 三、为什么用(收益)
- 提升转化率(亚马逊官方常引用:平均可提升数个百分点的销量)。
- 降低退货/差评(图文讲清卖点与使用方法)。
- 强化品牌形象;移动端展示更友好。

## 四、高级 A+ 的模块(本工具全支持,共 19 种)
图片/图文:完整图片、单图文、双图、四图、背景图片;
文字/结构:文本、问答、技术规格;
轮播:简单/规则/导航/视频图像;视频:全视频、含文本视频;
对比表 1/2/3(按 ASIN 对比);热点 1/2(图上打点标注)。
→ 每个模块要填什么/尺寸/字数,见 `modules.md`;可直接照抄的 spec 见 `spec-examples.md`。

## 五、内容要求(常见)
- 一篇 A+ **最多 7 个模块**。
- 文案有字数上限(各模块不同,见 modules.md);图片有最小尺寸要求。
- **语言**要符合站点(美国站=英文);可为不同 ASIN/站点分别制作。
- 文案/图片须真实合规:不得有**保证性/绝对化用语、竞品贬低、价格促销、联系方式/外链、版权侵权**等(否则审核驳回)。
- **AI 生成图片需按亚马逊要求做"AI 披露"**(见 SKILL.md 的 AI 披露说明)。

## 六、流程(本工具里)
1. 运营给素材文件夹 → 2. Claude Code 看图、定排版(见 `layout-planning.md`)→ 3. 生成文案 → 4. 调接口建**草稿** → 5. 运营在后台**预览**确认 → 6. 运营**手动提交审核** → 7. 通过后上线。
> 本工具只产出**草稿**,绝不自动提交;提交与合规确认由运营负责。

## 七、常见问答
- **A+ 和 listing 标题/五点有什么区别?** 标题/五点是文字字段;A+ 是详情页下方的图文增强区,需单独制作+审核。
- **做了就一定上线吗?** 不,需审核通过;违规会被驳回,改后可重提。
- **能改吗?** 可以,草稿可反复改;已发布的也可改版重提审核。
- **为什么不用 SP-API 自动建?** SP-API **不支持创建高级 A+**,所以本工具用紫鸟浏览器自动化在后台操作。
