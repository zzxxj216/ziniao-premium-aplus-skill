# 支持的模块 — 选型指南 + 每个要填什么

> `texts`/`bodies` 按**从上到下的字段顺序**填入(positional)。下面给出每个模块的顺序与字数上限。
> 图片尺寸是后台标注的**最小**要求,比它大可以(可能按比例显示/裁切)。
> **每张图建议给 `alt` 描述**(亚马逊算法/无障碍/审核用);每个图位支持**桌面 + 移动**两张图(`{"desktop":桌面,"mobile":移动,"alt":描述}`,旧字段 `image` 仍兼容,移动建议 600×450,不给则复用桌面)。标题等非必填项不给即留空。详见 spec-examples.md。

## 选型指南(按素材判断)
| 素材长这样 | 选 |
|---|---|
| 整幅带文字的横幅大图 | `完整图片` |
| 一张图 + 一段卖点文案 | `单图文` |
| 两张图并排(对比/两个卖点) | `双图` |
| 四个小图各配一句卖点 | `四图` |
| 纯文字段落(无图) | `文本` |

## 各模块字段

### 完整图片(高级完整图片)
- `images`: 1 张(桌面 ≥ 1464×600;可另给移动图 ≥ 600×450,不给则程序用桌面图生成移动图)
- `texts`: [标题 ≤80](可选)
- `bodies`: [正文 ≤300](可选)

### 单图文(带文本的单张高级图片)
- `images`: 1 张(≥ 800×600)
- `texts`: [副标题 ≤40, 标题 ≤80] —— **顺序:副标题在前、标题在后**(按后台 DOM 顺序)
- `bodies`: [正文 ≤500]

### 双图(包含文本的高级双图片)
- `images`: 2 张(各 ≥ 650×350,左、右)
- `texts`: [标题1 ≤50, 标题2 ≤50](左、右)
- `bodies`: [正文1 ≤300, 正文2 ≤300](左、右)

### 四图(高级四图片和文本)
- `images`: 4 张(各 ≥ 300×225)
- `texts`: [模块标题 ≤80, 小标题1 ≤30, 小标题2, 小标题3, 小标题4]
- `bodies`: [正文1 ≤150, 正文2, 正文3, 正文4]

### 文本(高级文本)
- `images`: []
- `texts`: [标题 ≤80]
- `bodies`: [正文 ≤5000]

### 背景图片(包含文本的高级背景图片)
- `images`: 1 张背景图(文字叠在图上)
- `texts`: [副标题 ≤40, 标题 ≤60]
- `bodies`: [正文 ≤300]
- 可选 `layout`: 图右(默认)/ 图左 —— 文字块左右位置
- 可选 `font_color`: 白(默认)/ 黑 —— 叠加文字颜色(按背景明暗选,保证可读)
- 适合:整幅氛围图 + 少量叠加文字

### 轮播(高级、简单的图像轮播)
- 用 `panels` 数组(2~6 个面板),每个面板独立一图一文:
  ```json
  {"type":"轮播","panels":[
     {"image":"/p/1.png","texts":["正文≤200"]},
     {"image":"/p/2.png","texts":["正文≤200"]}
  ]}
  ```
- 每面板图 ≥1464×600;程序逐面板:点面板标签→传图→填文。按钮/ASIN 跳转为可选(暂不填)。

### 问答(高级问答)
- `images`: []
- `texts`: 按 **问题, 回答, 问题, 回答…** 顺序排(2~5 组 = 4~10 个);程序自动点「添加问题」加够行
  - 问题 ≤120,回答 ≤250
- 例:`["是否防水?","防水且可机洗","几种图案?","10 种恐龙图案", ...]`

### 技术规格(高级技术规格)
- `images`: []
- `texts`: **[标题, 规格1, 定义1, 规格2, 定义2, …]**(4~16 组规格;程序自动点「添加规格」加够行)
  - 规格 ≤30,定义 ≤500,标题 ≤80
- 可选 `layout`:单表(默认)/ 两表(仅桌面端排布不同)

### 视频(高级全视频)
- `video`: mp4 绝对路径(服务端机器可访问);上传后亚马逊会处理一段时间
- `texts`: [标题 ≤80];`bodies`: [正文 ≤300]

### 热点(高级热点1 / 热点2)
- `images`: 1 张底图(1464×600)
- `hotspots`: `[{"title":"...","body":"..."}, ...]` —— 程序在图上按坐标铺开放置每个热点并填标题/正文
- 热点2 另可 `texts`/`bodies`(模块级标题/正文)

### 对比表(高级比较表 1 / 2 / 3)
三种版式,都靠 `asins`(真实 ASIN,须该店铺可用)+ `images`(各产品图)+ `texts`(标题):
- **对比表1**:主角商品(asins[0]=主角,图 200:225)+ 多个比较产品 + 比较指标
- **对比表2**:多产品并列,各 产品图(300:225)+ `img_titles`(图片标题)+ ASIN + 功能比较
- **对比表3**:主角(竖图 488:700)+ 比较产品
- ⚠️ ASIN 必须真实有效,否则保存/提交报错;`asins` 顺序对应产品槽位
- 完整 spec 见 `spec-examples.md`

## 样式开关(模块底部的成对图标,可选)
这些只改外观、有默认值,通过 spec 的可选字段设置(本质是选底部那对图标的 [0]/[1]):
- **单图文 / 双图 — 图文左右**:`layout: "图右"`(默认,文左图右) / `"图左"`(图左文右)
- **高级问答 — 背景**:`background: "白"`(默认) / `"黑"`(黑底白字)
- **背景图片 — 文字样式**:`layout: "图右"`/`"图左"` 控制文字块左右,`font_color: "白"`/`"黑"` 控制叠加文字颜色
- **高级技术规格 — 表格排布**(仅桌面端有别,手机端相同):
  - 默认 = **单个表格**(N 行排成一列)
  - 另一种 = **拆成两个表格**(各放一半行,桌面并排显示)

## 提醒
- 文案要符合所在站点语言(美国站=英文)。
- 图片须放在**中心服务那台机器**能读到的路径。
- 文案超限会被程序按 maxlength 截断;最好你生成时就控制在上限内。
- **推荐用按字段名填充**(`title`/`subtitle`/`body`/`nav`/`img_title`,见 api.md):按字段 placeholder 精确填、不怕顺序错,且中英双语 placeholder 都认。
  positional 的 `texts`/`bodies` 仍兼容(作兜底)。
