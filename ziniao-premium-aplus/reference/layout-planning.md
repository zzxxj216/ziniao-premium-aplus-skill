# 排版规划:从"一个文件夹"到"确认好的排版方案"

> 解决的问题:运营丢来一个素材文件夹,你(Claude Code)怎么决定**用哪些模块、按什么顺序、配什么文案**,并在**动手建之前和运营确认排版**。
> 原则:**你来识别+选型+排版(你能直接看图),程序只负责执行创建**。建完只是草稿,运营再预览确认。

## 步骤

### 1. 盘点 + ⭐先判断:每张图是"成品横幅"还是"原始照片"(最关键!)
**逐张真的"看"图**(用 Read 打开,你能看到内容),先给每张图归类——这一步决定选型对不对:

| 类别 | 特征 | 怎么用 |
|---|---|---|
| **A. 成品横幅**(文字/排版已印在图里) | 图上已有标题、卖点文字、品牌名、完整设计(常见于 AI 生成的整张 A+ 图,宽幅) | **每张 = 1 个「完整图片」**(图自带文字,**不要再填标题/正文**,只给 `alt` 描述);多张用**轮播**打包(见第 2 步) |
| **B. 原始照片**(只有产品/场景,无文字) | 干净产品图、场景图、白底图,没有叠字 | 用 **单图文/双图/四图/背景图片**,**由你写文案** |
| **C. 信息图/对比** | 参数表、对比 | 文本/技术规格;按 ASIN 对比→对比表(需 ASIN) |
| **D. 视频** | .mp4 | 全视频 / 含文本视频 / 视频图像轮播 |

> ❗最常见错误:把 **A 类成品横幅** 塞进 单图文/四图、或给它再写标题 → 重复错乱。**成品横幅就用「完整图片」或「轮播」,只配 alt,不另写文字。**
> 量尺寸(可选):`sips -g pixelWidth -g pixelHeight <文件>`。**图太大不用手动改,程序上传前会自动缩放。**

#### ⭐ 图不满足要求 → 重绘(若你的 agent 支持绘图,如 Codex)
当某张图**不达标**就别硬用,先和运营确认后**重绘一张合规的**:
- 触发场景:**尺寸不够**(小于该模块最小值,放大会糊)、**比例严重不符**、**质量差/糊**、或**该位置缺图**。
- 怎么做:用你的**图像生成能力**(Codex 自带绘图)按**该模块的目标尺寸**(见 modules.md,如完整图片 1464×600、移动 600×450)生成,内容要**贴合该商品/品牌/卖点**(参考运营给的其它图风格)。
- 规矩:**先告诉运营"这张不达标,建议重绘成 XxX",得到同意再画**;别擅自替换运营的品牌成品图。AI 生成的图记得在 spec 里标 `"ai_generated": true`(走 AI 披露)。
- 画好后当作普通素材:远程服务先 `/aplus/upload` 上传,再用返回路径建。

### 2. ⭐图多怎么办:用轮播打包,守住"一篇最多 7 个模块"
一篇 A+ **最多 7 个模块**。成品横幅很多时(8~12 张),**别想塞成 8 个完整图片(超限)**,用**轮播**把多张打进一个模块:
- **一个「简单轮播」= 1 个模块,可装 2~6 张图**(规则/导航轮播 2~5 张)。
- 决策(N = 成品横幅数量):

| 图数 N | 怎么排 |
|---|---|
| 1~6 | 全用「完整图片」(每张一个);或前几张完整图片 + 其余打一个轮播 |
| 7 | 7 个完整图片(正好上限)|
| 8~12 | **分组**:几张关键的用完整图片,其余按主题打成 1~2 个简单轮播(每个≤6 张),总模块数 ≤7 |
| >12 | 选最重要的 ~6 个主题,每主题一个轮播(每个≤6张),其余舍弃 |

例:**10 张成品横幅** → `完整图片(主横幅)+ 轮播[3张:玩法步骤]+ 完整图片(卖点)+ 轮播[3张:使用场景]+ 完整图片(收尾)` = **5 个模块,装下 9~10 张**。

**原始照片(B 类)**才按内容选:一图配文→单图文;左右两图→双图;四个小卖点→四图;氛围叠字→背景图片;图上标注多个卖点→热点;按 ASIN 对比→对比表;纯文字/问答/参数→文本/问答/技术规格。

### 3. 排顺序(讲一个故事)
高级 A+ **一篇最多 7 个模块**。常用叙事结构(按需裁剪):
1. **开场横幅**(完整图片/背景图片):品牌+核心卖点一句话
2. **2~4 个卖点**(单图文/双图/四图/热点):逐个讲功能/材质/使用
3. **对比/参数**(对比表/技术规格):为什么选我们
4. **场景/情感 + 收尾**(背景图片/轮播/视频):使用场景、信任背书
5.(可选)**问答**:打消顾虑

### 4. ⭐ 和运营确认排版(动手前的关口)
把方案**结构化列给运营**,等确认/调整后再建。建议给这两样:

**(a) 模块清单**(图→模块→理由;成品横幅只配 alt、不写标题):
```
素材:10 张成品横幅(文字已印)→ 5 个模块(≤7,OK):
1. 完整图片  ← banner1.png            主横幅(自带文字)  alt:"建筑主题姓名贴,防水耐用"
2. 轮播简单  ← b2,b3,b4.png (3张)     玩法步骤          每张配 alt
3. 完整图片  ← b5.png                 卖点横幅          alt:"贴在衣服鞋子水壶上"
4. 轮播简单  ← b6,b7,b8.png (3张)     使用场景          每张配 alt
5. 完整图片  ← b10.png                收尾横幅          alt:"为小建筑迷打造"
(都是成品横幅 → 不写标题/正文,只给 alt;图太大程序自动缩放)
```

**(b) ASCII 排版草图**:
```
┌──────────── 完整图片:主横幅 ────────────┐
├──────── 轮播(3张):玩法步骤 ─────────────┤
├──────────── 完整图片:卖点 ──────────────┤
├──────── 轮播(3张):使用场景 ─────────────┤
├──────────── 完整图片:收尾 ──────────────┤
└──────────────────────────────────────────┘
```
(若是 B 类原始照片,则换成 单图文/四图 等并由你写文案——见第 2 步末尾)
然后问运营:**"这个排版顺序/选型 OK 吗?文案要不要改?缺的 ASIN/视频给一下?"** —— 得到确认(或按其反馈调整)后再进入第 5 步。

> 缺素材(对比表缺 ASIN、视频缺文件、图尺寸不够)就在这一步**回去问运营,别编造**。

### 5. 建草稿
按确认后的方案构造 spec(用按字段名 `title/subtitle/body`,见 api.md / spec-examples.md),调 `POST /aplus/create`。

### 6. 最终视觉确认(建完之后)
- 把返回的 `url` 给运营;让运营在 Seller Central 编辑器点 **Preview** 看实际效果(这是排版的最终确认)。
- 不满意 → 按反馈改 spec 重建(草稿可反复改)。
- 满意 → 运营**手动提交审核**(并确认 AI 披露等合规项)。

## 小结:"确认排版"分两层
1. **建之前**:你给"模块清单 + ASCII 草图",运营文字确认(快、零成本、可改)。
2. **建之后**:运营在后台 **Preview** 看真实渲染(最终拍板)。
两层之间你可以多次迭代,直到运营满意再提交。
