# create_aplus —— 全部 19 个模块的完整 spec 示例

> 调用:`POST {APLUS_BASE_URL}/aplus/create`,头 `X-API-Key`。
> 顶层:`{"store": "店铺名(可选)", "name": "内容描述名称(必填)", "modules": [ ...下面任意组合... ]}`
> **`modules` 可放多个(任意类型混合,按顺序排版进同一篇);一篇最多 7 个模块(亚马逊上限),勿超。**
> 图片/视频路径必须是**中心服务那台机器**可访问的绝对路径。文本超 maxlength 会被截断。

## 图片格式(重要:alt + 桌面/移动两张图)
图片项有两种写法:
- 简写:`"images": ["/d1.png"]`(仅桌面图,alt 自动用内容名兜底)
- **完整写法(推荐)**:每张图给 alt 描述、可加移动图
  ```json
  "images": [
    {"desktop":"/desktop.png", "mobile":"/mobile.png", "alt":"A child's water bottle with a dino name label"}
  ]
  ```
  - `desktop`:桌面端图;旧字段 `image` 仍兼容,两者等价
  - `alt`:**图片描述,务必填**(亚马逊算法/无障碍会用;也利于审核与 SEO)。**上限 100 字符**,超长会被截断
  - `mobile`:移动端图,不给则移动端复用桌面图
  - **图太大会自动等比缩小**(不裁剪、整图保留;按模块推荐宽度,如横幅 1464、移动 600);不想自动缩放给模块 `"resize": false`
  - 源文件不会被修改(缩放结果写到服务端缓存)
- 单图模块也可直接写顶层:`{"type":"完整图片","desktop":"/d.png","mobile":"/m.png","alt":"..."}`
- 轮播面板同理:`{"desktop":"/d.png","mobile":"/m.png","alt":"...","texts":["可选标题"]}`
- 也可用 `"alts":["描述1","描述2"]`(与 images 顺序对应)或模块级 `"alt":"统一描述"`。
- 标题/副标题等**非必填**:不给就留空(但 **alt 建议每张都给**)。

```json
{"type":"完整图片","images":["/p/banner.png"],"texts":["标题≤80"],"bodies":["正文≤300"]}
```
```json
{"type":"单图文","images":["/p/a.png"],"texts":["副标题≤40","标题≤80"],"bodies":["正文≤500"],"layout":"图右"}
```
```json
{"type":"双图","images":["/p/l.png","/p/r.png"],"texts":["标题1≤50","标题2≤50"],"bodies":["正文1≤300","正文2≤300"]}
```
```json
{"type":"四图","images":["/p/1.png","/p/2.png","/p/3.png","/p/4.png"],
 "texts":["模块标题≤80","小标题1≤30","小标题2","小标题3","小标题4"],
 "bodies":["正文1≤150","正文2","正文3","正文4"]}
```
```json
{"type":"背景图片","images":["/p/bg.png"],"texts":["副标题≤40","标题≤60"],"bodies":["正文≤300"],
 "layout":"图左","font_color":"黑"}
```

## 轮播(variant:简单/规则/导航/视频图像)

```json
{"type":"轮播","variant":"简单","panels":[
  {"image":"/p/1.png","texts":["正文≤200"]},
  {"image":"/p/2.png","texts":["正文≤200"]}
]}                                                            // 简单:2~6 面板
```
```json
{"type":"轮播","variant":"规则","title":"模块标题≤100","panels":[
  {"image":"/p/1.png","texts":["面板文本≤20","导航文本≤25"]},
  {"image":"/p/2.png","texts":["面板文本","导航文本"]}
]}                                                            // 规则/导航:2~5 面板,每面板必填导航文本(texts 末位)
```
```json
{"type":"轮播","variant":"视频图像","panels":[
  {"image":"/p/1.png","video":"/v/1.mp4","texts":["副标题≤50","正文≤500"]},
  {"image":"/p/2.png","video":"/v/2.mp4","texts":["副标题","正文"]}
]}                                                            // 每面板图+视频,2~6 面板
```

## 对比表(1/2/3)——只填 ASIN,图片+标题自动拉取
**核心机制**:填了 ASIN,亚马逊会**自动拉取该商品的图片和标题**——所以**不用给图片**。但自动拉来的**标题太长**(超字段上限),程序会**清空后填你给的短标题 `names`**(对比表2 ≤30、对比表1/3 ≤25)。

```json
{"type":"对比表2","title":"为什么选我们",
 "asins":["B0XXXXXXX1","B0XXXXXXX2","B0XXXXXXX3"],   // 各比较产品的真实 ASIN
 "names":["我们的款≤30","普通款","竞品"]}             // 短标题,替换自动拉取的长标题(不给则保留长标题)
```
```json
{"type":"对比表1","title":"对比","asins":["B0主角","B0产品2","B0产品3"],"names":["主角款≤25","款2","款3"]}
```
```json
{"type":"对比表3","title":"对比≤25","asins":["B0主角","B0产品2"],"names":["主角款≤25","款2"]}
```
> - ASIN 要**真实**(亚马逊按它拉图+标题);产品数按 asins 顺序,更多产品在后台再"添加商品"。
> - `names`=短标题(必给,否则标题过长);`img_titles` 是旧别名,等价。
> - 一般**不用给 `images`**(自动拉取);确需自定义产品图才给 `images`。

## 热点类(1/2,图上坐标标注)

```json
{"type":"热点1","images":["/p/scene.png"],
 "hotspots":[{"title":"卖点1≤80","body":"说明1≤300"},{"title":"卖点2","body":"说明2"}]}
```
```json
{"type":"热点2","images":["/p/scene.png"],"texts":["模块标题≤80"],"bodies":["模块正文≤300"],
 "hotspots":[{"title":"卖点1","body":"说明1"},{"title":"卖点2","body":"说明2"}]}
```
> 热点在底图上**横向均匀铺开**放置;每个热点有标题+正文(还可在后台单独加热点小图)。

## 样式开关(可加在支持的模块上,可选)
- `layout`: `图右`(默认)/`图左` —— 单图文/双图/背景图片
- `font_color`: `白`(默认)/`黑` —— 背景图片(叠加文字色);问答可用 `background`
- `layout`: `单表`/`两表` —— 技术规格(仅桌面端排布)
